🎨 CSS Combinators Masterclass

Learn how to target elements with precision using CSS combinators

1. Descendant Combinator (Space)

Syntax: ancestor descendant
Targets: ALL descendants (children, grandchildren, etc.) of the ancestor element
.container div { background: #ff7675; }
✅ Direct child div - STYLED
✅ Nested div inside section - STYLED
✅ Deeply nested div - STYLED

❌ Paragraph - NOT STYLED

2. Child Combinator (>)

Syntax: parent > child
Targets: ONLY direct children, not deeper descendants
.container > p { background: #00b894; }

✅ Direct child paragraph - STYLED

✅ Another direct child paragraph - STYLED

❌ Nested paragraph - NOT STYLED (not direct child)

❌ Deeply nested paragraph - NOT STYLED

3. Adjacent Sibling Combinator (+)

Syntax: element + next-sibling
Targets: The FIRST sibling element that immediately follows
h2 + p { background: #e84393; }

🎯 This heading triggers the style

✅ This paragraph immediately follows h2 - STYLED

❌ This paragraph doesn't immediately follow h2 - NOT STYLED

🎯 Another heading

❌ This div follows h2, not p - NOT STYLED

❌ This p doesn't immediately follow h2 - NOT STYLED

🎯 Third heading

✅ Again, immediate sibling paragraph - STYLED

4. General Sibling Combinator (~)

Syntax: element ~ sibling
Targets: ALL following siblings of the same type (not just the first one)
.trigger ~ .target { background: #6c5ce7; }
❌ Before trigger - NOT STYLED
🎯 TRIGGER ELEMENT
✅ First target after trigger - STYLED
❌ Not a target class - NOT STYLED
✅ Second target after trigger - STYLED

❌ Different element type - NOT STYLED

✅ Third target after trigger - STYLED

📚 Quick Reference Summary

🌳 Descendant (space)

.parent .child

Selects ALL nested elements

👶 Child (>)

.parent > .child

Selects DIRECT children only

👯 Adjacent Sibling (+)

.first + .second

Selects NEXT sibling only

👨‍👩‍👧‍👦 General Sibling (~)

.first ~ .others

Selects ALL following siblings
